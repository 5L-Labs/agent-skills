# Paper Summary Workflow

## When to use this workflow
When you need to quickly understand and summarize academic papers, especially ML/AI papers, to determine their relevance and extract key claims.

## Steps
1. Fetch the paper using arxiv ID or URL
2. Read the abstract and introduction to understand the core problem
3. Look for:
   - Main contribution/claim
   - Key experiments/results
   - Specific numbers or benchmarks mentioned
   - Limitations or nuances discovered
4. Verify any summaries against the actual paper text
5. Note if the summary conflates multiple papers or misrepresents findings
6. Extract any surprising or counterintuitive findings

## Pitfalls to avoid
- Assuming a summary is accurate without verification
- Conflating multiple papers with similar topics or numbers
- Missing important nuances or limitations in the paper
- Overlooking the actual scope (e.g., specific hardware, model sizes, datasets)

## Verification
- Check that numbers in summaries match those in the paper
- Verify that claimed performance is for the specific conditions mentioned
- Note any "anomalies" or unexpected results the authors highlight
